# SRS Stage 6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 將 SRS 從 6 個階段（0–5）擴充為 7 個階段（0–6），新增 30 天等待期的 stage 5，stage 6 為畢業；同時將資料庫中現有 stage 5 的已畢業單字重置回 stage 0。

**Architecture:** 修改 `lib/srs.ts` 的間隔陣列（5 個 entries，最後一個改為 30 天）與畢業判定邏輯（graduation 從 stage 5 改為 stage 6），更新所有參照 `reviewStage === 5` 作為畢業判斷的程式碼改為 `=== 6`，最後透過 Neon MCP 執行 SQL 重置現有已畢業單字。

> **注意：** `INTERVALS_MS[stage]` 是在 review stage N 時用來排程下一次複習的等待時間。因此 30 天的等待是 `INTERVALS_MS[4]`（reviewing stage 4 → stage 5, wait 30 days）。`INTERVALS_MS` 只需 5 個 entries（不是 6 個），不可加入第 6 個 entry，否則會是 dead code（graduation early return 在 `INTERVALS_MS[stage]` 之前發生）。

**Tech Stack:** Next.js App Router, Drizzle ORM, Neon serverless PostgreSQL, TypeScript

### Task 1: 更新 SRS 核心邏輯

**Files:**
- Modify: `lib/srs.ts`
- Modify: `lib/types.ts`

- [ ] **Step 1: 修改 `lib/srs.ts`**

  將 `INTERVALS_MS[4]` 從 28 天改為 30 天（保持 5 個 entries），並將 graduation 判定從 `nextStage === 5` 改為 `nextStage === 6`，`Math.min` 上限從 5 改為 6，return type 加入 `| 6`：

  > **關鍵設計：** `INTERVALS_MS[stage]` 在 reviewing stage N 時被使用，用來排程下一次複習。
  > - Reviewing stage 4 → `INTERVALS_MS[4]` = **30 天** → stage 5 due in 30 days ✓
  > - Reviewing stage 5 → `nextStage === 6` → graduation (Infinity)
  > - 不需要第 6 個 entry；若加入 `INTERVALS_MS[5]` 永遠不會被執行（dead code）。

  ```ts
  const INTERVALS_MS = [
    1  * 24 * 60 * 60 * 1000,
    3  * 24 * 60 * 60 * 1000,
    7  * 24 * 60 * 60 * 1000,
    14 * 24 * 60 * 60 * 1000,
    30 * 24 * 60 * 60 * 1000,  // stage 4→5: 30-day wait before graduation review
  ]

  export function getNextReviewAt(stage: number, remembered: boolean): { stage: number; nextReviewAt: number } {
    if (!remembered) {
      return {
        stage: 0,
        nextReviewAt: Date.now() + INTERVALS_MS[0],
      }
    }
    const nextStage = Math.min(stage + 1, 6) as 0 | 1 | 2 | 3 | 4 | 5 | 6
    if (nextStage === 6) {
      return { stage: 6, nextReviewAt: Infinity }
    }
    const next = new Date(Date.now() + INTERVALS_MS[stage])
    next.setHours(0, 0, 0, 0)
    return {
      stage: nextStage,
      nextReviewAt: next.getTime(),
    }
  }
  ```
